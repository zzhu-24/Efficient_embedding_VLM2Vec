## 利用 `compute_head_importance.py` 进行注意力头与 MLP 剪枝的说明

本说明文档解释：

- `compute_head_importance.py` 如何计算 **注意力 KV 组（head）** 和 **MLP 中间神经元** 的重要性；
- 这些重要性结果如何通过 `train_retrieval_rmv_layer.sh` 中第 36–53 行的环境变量配置，参与到实际训练过程中的 **剪枝策略**。

---

## 一、`compute_head_importance.py` 做了什么？

### 1. 基本思路：Taylor 重要性

脚本使用论文 *Michel et al., 2019* 中的 **Taylor 重要性** 作为重要性度量：

\[
\text{importance} = \mathbb{E}\left[\,|\text{activation} \times \text{gradient}|\,\right]
\]

在代码中：

- 对 **注意力模块**：在每一层 self-attention 的 `o_proj` 输入处挂前向/反向 hook，得到每个 **Q 头** 的激活和梯度，并按 **KV group（GQA 的 KV 组）** 汇总；
- 对 **MLP 模块**：在每一层 MLP 的 `down_proj` 输入处挂 hook，对每个 **中间层神经元** 统计 `|act * grad|`，对 batch 和 seq 求和，得到每一层内所有神经元的重要性。

### 2. 校准数据 & 模型加载

脚本会：

- 使用与训练相同的 `MODEL_NAME`、`dataset_config` 和 `data_basedir` 加载 **相同的数据处理与数据格式**；
- 构建与训练中一致的 `MMEBModel`，支持：
  - `--model_name`
  - `--checkpoint_path`（可选，加载 LoRA 权重）
  - `--delete_L` / `--delete_n`（与训练时的层删除配置保持一致）；
- 从指定的 `dataset_config` 中取第一个数据集，构建一个小规模 **校准 DataLoader**，运行若干个 batch（由 `--num_calibration_batches` 和 `--batch_size` 控制）；
- 对每个 batch：
  - 前向得到最后一层 hidden states；
  - 用最后一个 token 的 hidden 做一个简单的 L2 范数 proxy loss；
  - 反向传播产生梯度，从而通过 hook 累积重要性。

### 3. 关键命令示例

在 `train_retrieval_rmv_layer.sh` 中已经给出一个典型示例（被注释掉）：

```bash
python compute_head_importance.py \
    --model_name $MODEL_NAME \
    --dataset_config "$DATASET_CONFIG_PATH" \
    --data_basedir "$DATA_BASEDIR" \
    --output_dir $EXP_DIR/importance \
    --delete_L 30 \
    --delete_n 23 \
    --num_calibration_batches 32 \
    --batch_size 4 \
    --bf16
```

运行完成后，`$EXP_DIR/importance` 目录下会生成注意力头和 MLP 的重要性文件及可视化图片。

## 二、`compute_head_importance.py` 的输出文件

脚本会在 `--output_dir` 下生成（只列出与剪枝直接相关的）：

- **注意力头重要性**
  - `head_importance.json`
  - 命名格式：`[delete_L]_[delete_n]_head_importance.json`，例如：
    - `"[36]_[0]_head_importance.json"`（对应 `delete_L=[36]`, `delete_n=[0]`）
  - JSON 结构大致为：
    ```json
    {
      "0": { "0": score_0_0, "1": score_0_1, ... },  // 第 0 层，每个 KV group 的重要性
      "1": { "0": score_1_0, "1": score_1_1, ... },
      ...
    }
    ```

- **MLP 神经元重要性**
  - `mlp_importance.json`
  - 命名格式：`[delete_L]_[delete_n]_mlp_importance.json`，例如：
    - `"[28]_[14]_mlp_importance.json"`
  - JSON 结构大致为：
    ```json
    {
      "0": [score_0_0, score_0_1, ...],  // 第 0 层所有中间神经元的重要性
      "1": [score_1_0, score_1_1, ...],
      ...
    }
    ```

- 另有若干可视化图片（heatmap / ranking / boxplot），用于人工分析，但不是训练脚本直接依赖的文件。

## 三、如何在 `train_retrieval_rmv_layer.sh` 中使用这些结果

`train_retrieval_rmv_layer.sh` 的第 36–53 行定义了与剪枝相关的环境变量：

```bash
export DATA_BASEDIR=/home/infres/zzhu-24/PRIM/VLM2Vec/experiments/public/data/vlm2vec_train/MMEB-train

# --- Head Pruning Config ---
# Option A: Provide a JSON file with per-layer KV group pruning spec
#   (generated by compute_head_importance.py or manually created)
#   Format: {"0": [2, 5], "3": [1, 7]}  means prune groups 2,5 from layer 0, groups 1,7 from layer 3
# export HEAD_PRUNE_CONFIG=/home/infres/zzhu-24/PRIM/VLM2Vec/importance_output/[36]_[0]_head_importance.json
#
# Option B: Uniform pruning - prune N least-important KV groups from every layer
# export HEAD_PRUNE_N=2

# --- MLP Pruning Config ---
# Ratio of MLP intermediate neurons to prune per layer (0.0 = no pruning)
export MLP_PRUNE_RATIO=0.5
#
# Path to mlp_importance.json (generated by compute_head_importance.py)
# Required when MLP_PRUNE_RATIO > 0 for importance-based pruning
export MLP_IMPORTANCE_PATH=/home/infres/zzhu-24/PRIM/VLM2Vec/importance_output_ckpt5500/[28]_[14]_mlp_importance.json
```

后面脚本会根据这些环境变量构造传给 `train.py` 的参数：

```bash
# --- Build head_prune arg ---
HEAD_PRUNE_ARG=""
if [ -n "${HEAD_PRUNE_CONFIG:-}" ]; then
    HEAD_PRUNE_ARG="--head_prune_config $HEAD_PRUNE_CONFIG"
elif [ -n "${HEAD_PRUNE_N:-}" ]; then
    HEAD_PRUNE_ARG="--head_prune_n $HEAD_PRUNE_N"
fi

# --- Build mlp_prune arg ---
MLP_PRUNE_ARG=""
if [ -n "${MLP_PRUNE_RATIO:-}" ]; then
    MLP_PRUNE_ARG="--mlp_prune_ratio $MLP_PRUNE_RATIO"
    if [ -n "${MLP_IMPORTANCE_PATH:-}" ]; then
        MLP_PRUNE_ARG="$MLP_PRUNE_ARG --mlp_importance_path $MLP_IMPORTANCE_PATH"
    fi
fi
```

最终在调用 `train.py` 时，这两个参数会被拼接到命令行中：

```bash
... train.py \
    ... \
    $HEAD_PRUNE_ARG \
    $MLP_PRUNE_ARG \
    --report_to wandb ...
```

### 1. 注意力头剪枝（Head Pruning）

有两种使用方式：

- **方式 A：基于 `HEAD_PRUNE_CONFIG` 的显式 KV 组列表**
  - 你可以根据 `head_importance.json` 中的重要性分数，自己选出要剪掉的 KV group，下成 JSON，例如：
    ```json
    {
      "0": [2, 5],
      "3": [1, 7]
    }
    ```
    表示：第 0 层剪掉 KV group 2 和 5，第 3 层剪掉 KV group 1 和 7。
  - 将该 JSON 路径赋值给 `HEAD_PRUNE_CONFIG`，脚本会传给 `train.py --head_prune_config`，在模型构建时根据这个配置屏蔽对应的 KV heads。

- **方式 B：基于 `HEAD_PRUNE_N` 的统一剪枝**
  - 若设置 `HEAD_PRUNE_N=N` 且不提供 `HEAD_PRUNE_CONFIG`，训练代码会在内部按照 `head_importance.json` 统计每一层的 KV group 重要性，并从 **全局最不重要** 或 **每层最不重要** 的 group 中统一剪掉 N 个（具体行为由模型内部实现决定）；
  - 核心思想依然是用 `compute_head_importance.py` 得到的分数，选择低重要性的 KV groups 进行剪枝。

### 2. MLP 剪枝（MLP Pruning）

MLP 剪枝由两个量共同控制：

- `MLP_PRUNE_RATIO`：每层需要剪掉的中间神经元比例（例如 `0.5` 表示剪掉 50% 的中间神经元）；
- `MLP_IMPORTANCE_PATH`：指向 `compute_head_importance.py` 生成的 `mlp_importance.json`，例如：
  - `/home/infres/zzhu-24/PRIM/VLM2Vec/importance_output_ckpt5500/[28]_[14]_mlp_importance.json`

在 `train.py` 中：

- 若 `--mlp_prune_ratio > 0`，则会尝试对每一层的 MLP 中间层进行剪枝；
- 若同时提供了 `--mlp_importance_path`，训练代码会：
  - 读取该 JSON；
  - 对每一层内部，根据重要性从小到大排序；
  - 按 `mlp_prune_ratio` 的比例，从**最不重要的神经元开始剪掉**；
- 若不提供 `--mlp_importance_path`，则可能回退为随机或不剪枝（具体行为取决于实现，一般建议始终提供 importance 文件）。

因此，`compute_head_importance.py` 里的 MLP 重要性分数 **直接决定了在每一层中哪些神经元会被优先裁剪**。

## 四、端到端使用流程总结

1. **离线计算重要性（一次性操作）**  
   在希望的模型配置（`MODEL_NAME`、`delete_L` / `delete_n` 等）和数据配置下，运行：
   - `python compute_head_importance.py ...`
   - 得到对应的 `head_importance.json` 与 `mlp_importance.json`。

2. **配置剪枝参数**  
   在 `train_retrieval_rmv_layer.sh` 中：
   - 若需要 head 剪枝：
     - 方式 A：根据 `head_importance.json` 写好 `HEAD_PRUNE_CONFIG` JSON，并导出其路径；
     - 或 方式 B：只设置 `HEAD_PRUNE_N` 为一个整数；
   - 若需要 MLP 剪枝：
     - 设置 `MLP_PRUNE_RATIO`（例如 `0.5`）；
     - 将 `MLP_IMPORTANCE_PATH` 指向对应的 `mlp_importance.json`。

3. **启动训练**  
   运行：
   ```bash
   bash experiments/public/train/train_retrieval_rmv_layer.sh
   ```
   在训练过程中：
   - 模型会根据传入的 `--head_prune_config` / `--head_prune_n` 和 `--mlp_prune_ratio` / `--mlp_importance_path`，利用事先计算好的重要性分数 **动态屏蔽/裁剪** 部分注意力头和 MLP 神经元；
   - 以实现参数高效化或进一步的压缩/正则化效果。

通过以上流程，可以做到：**先用少量校准数据估计各层各子模块的重要性，再在正式训练中优先保留高重要性的结构、裁剪低重要性的结构**。
